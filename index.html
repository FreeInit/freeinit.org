FreeInit.org is the community initiative in support of free and open-source alternatives to the X Desktop Group's systemd init system and middleware suite.

FreeInit.org is not an init system itself; it is a movement in support of projects aiming to build alternatives to systemd's init system and other systemd components.

Our initial goals are:

  1. Creating a list of systemd components, the FOSS alternatives to these, and the status and systemd-compatibility of these alternatives.
    It will serve as a helpful resource both to users and interested developers.
  2. Creating a libsystemd implementation providing the various utility libraries (e.g. libsystemd-bus), and with init, journal, and login management functionality abstracted so that it may work with alternatives.
    This will unify the disparate libsystemd implementations of e.g. Basu and elogind.
  3. Promoting the uptake of seatd/libseat, respectively a seat-management daemon and a library providing an abstract interface backed by either seatd or systemd-logind/elogind.
    Direct dependency on the systemd-logind interface would thereby be minimised.
  4. Building solidarity between those who think free software is more than just a buzzword.

The FreeInit.org initiative is young and any support is welcomed. If you are interested, please visit our GitHub repository and create an issue.

Our members projects include:

  • InitWare: A fork of systemd's init system aiming for modularity, portability to BSD, and reduced project scope.
  • SharpInit: A modular, high-performance init daemon in C#.
